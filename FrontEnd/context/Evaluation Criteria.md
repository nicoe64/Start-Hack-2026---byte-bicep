# Evaluation Criteria

How hackathon submissions for the START Hack 2026 [[Studyond]] challenge will be judged. There are no rigid rubrics — submissions can lean technical, design-focused, or conceptual.

## What Matters

### 1. Real Problem Solving

Does the solution address genuine student pain points in the [[Thesis Journey]]? Not hypothetical problems, but the actual friction: choosing the wrong topic, finding no supervisor, lacking structure, feeling isolated. The best entries will show evidence of understanding what thesis students actually struggle with.

### 2. Creativity

Novel approaches matter. Going beyond obvious solutions — beyond "just a chatbot" — into new interaction paradigms, unexpected combinations, or reframed problem definitions. What has the team seen that others have not?

### 3. Coherence

Does the experience make sense as a whole? A well-integrated flow that handles three use cases coherently beats a scattered prototype that touches ten. The pieces should fit together into something a student would actually want to use.

### 4. Ecosystem Usage

Does the solution leverage [[Studyond]]'s existing building blocks? The [[Data Model|mock data]] (topics, supervisors, companies, experts, students), the [[Platform Overview|platform features]] (matching, messaging, applications), and the [[Three-Sided Marketplace|three-sided model]] (students, companies, universities) are there to be used. Solutions that ignore the ecosystem and build from scratch miss the point.

### 5. Compelling UX

Built on [[Editorial Minimalism]] principles: content-first, quiet confidence, functional color, consistent rhythm, progressive disclosure. Strong interaction design that feels considered, not default. Typography, spacing, and motion should reflect the Studyond brand language.

## Submission Formats

All are valid: a beautifully designed prototype with no code, a working agent system with rough UI, a novel conceptual framework with mockups. Quality and creativity matter more than technical depth. See [[Challenge Brief]] for the full problem statement and [[Possible Directions]] for starting points.
